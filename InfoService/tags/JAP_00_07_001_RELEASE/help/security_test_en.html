Checking settings of browser / filter

We will use test sites which let us know whether your browser (and filter proxy, if any) is correctly configured.
You may try both sites or choose one.

Seite 1
This site (leader.ru) shows a quick overview of all the HEADER information that your browser gives to the site. The topic Collected Information shows this information. In order to see what effect the settings bring about, the following preceeding is recommended:

  1. At the beginning, visit the site without or with deactivated JAP and filter proxy resp. correct data protection settings for your browser. Write down the following values (or memorize them):

    1. Reported remote address (shows your external IP address)
    2. Browser (exact name of your browser including versioning number)
    3. OS (your operating system)
    4. Referer (The site where you came from, this might be not displayed)
    And now have a look at the information gathered about your system under the topic Additional Information.

  2. Now reload the site with activated JAP (and filter proxy).
    1. This should not be the same as before.
    2. Data should be faked (Firefox -> Opera) or significantly less detailed.
    3. Same here.
    4. Referer should either be missing or showing the currently displayed site, not however the site you came from.
    The information from Additional Information should be gone almost entirely.
Seite 2
This site (gurushaven.de) also performs some tests and explains them (german language). In the left menu, click Anonymitäts-Test and then the subentry Zum Anonymitäts-Test. This site, too, should be visited with JAP and Filterproxy resp. correct data protection settings for your browser disabled once and then enabled in order to see the difference.
The following should be different:
  • Detaillierte Anzeige Deiner Bildschirmeinstellungen: The values of screen resolution etc should be incorrect.
  • Anzeige Deiner (öffentlichen) IP-Adresse: The shown IP address should not match your own external IP address.
  • Anzeige Deiner "echten", privaten IP Adresse trotz NAT an der Firewall: This Java applet should have been deactivatet. When using proxomitron, a corresponding message will occur.
  • Anzeige des Referers (Referrer): Referer should be "http://www.gurusheaven.de/security/anonymitaets_test.shtml". No referer at all is also ok but might cause trouble on certain sites in rare cases.
In case the results of the test do not meet requirements this might be due to the following reasons:
  • JAP has not been installed correctly
  • Not have not set the data protection configuration for your browser correctly
  • No filter proxy has been used.
  • Filter proxy is not using correct filter settings.
  • Filter proxy is not using JAP as external proxy.
  • Your browser is not using the filter proxy as proxy.
Further test sites:

  • Astalavista Displays connection information (HTTPS)
  • Proxyjudge Attempts to make a statement about anonymity level (improper rating scheme)
