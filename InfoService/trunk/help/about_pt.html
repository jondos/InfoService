Sobre o Java Anon Proxy

Java Anon Proxy (JAP)

Java Anon Proxy (JAP) possibilita navegar na web sem ser observado. Isto quer dizer que nem o servidor pedido nem nenhum servidor na Internet pode saber que utilizador viu determinada página.

JAP tem de ser instalado no computador do utilizador para lhe permitir navegar anonimamente sem ser observado. Isto é necessário porque todos os pedidos e todas as respostas dos servidores são enviadas, não directamente para o servidor web, mas através de uma Mistura em Cascata. Estas misturas são estações interpostas, através das quais todas as mensagens são enviadas.

Como muitos utilizadores usam o seviço Anon ao mesmo tempo, as ligações de internet de cada um ficam escondidas no meio das outras; Cada utilizador é responsável pela sua ligação. Ninguém, de fora, outro utilizador ou mesmo o operador do serviço Anon pode determinar que ligações de internet foram requisitadas por um determinado utilizador.

O Serviço Anon

Em geral, pelo menos duas misturas de proxies trabalham no serviço anon, cada uma delas operada por uma instituição independente que declarou obrigatoriamente não armazenar ficheiros log das ligações transportadas nem trocar dados sensíveis com outros operadores de proxy.

Contudo, como não se deve confiar plenamente em nenhuma das estações interpostas todos os dados enviados são encriptados várias vezes (uma por estação de mistura).

A encriptação dos pedidos e a desencriptação das respostas ocorrem no programa JAP. O utilizador apenas tem de configurar o seu browser de forma que todos os pedidos sejam direccionados para o JAP.

O anonimato do Utilizador ocorre através do envio de cada pedido por multi-encriptação através de uma série de estações interpostas baseadas num conceito de mistura de David Chaum. Mesmo que, por absurdo, o comportamento de todos os utilizadores fosse igual (enviando o a mesma quantidade de dados por unidade de tempo através da mesma mistura em cascasta), desde que apenas uma mistura não cooperasse com um atacante global, não seria possível reconstruir as acções de um utilizador individual.

O nosso objectivo é fornecer um serviço de acordo com esses critérios.

Atenção! A actual versão ainda não atingiu esses níveis de segurança, porque o software está em desenvolvimento! Contudo, esta versão já garante um padrão de segurança muito mais elevado do que os sistemas que trabalham com um único servidor proxy.

Esta versão JAP fornece protecção contra atacantes locais (i.e. o ISP, servidor web, o seu chefe ou patrão). Além disso, fornece protecção contra os operadores de mistura. Dependendo da mistura em que um operador opera, ele só pode obter informação sobre que endereço IP usa o serviço ou que pedidos foram enviados para a web, mas nunca a combinação das duas possibildades.

A actual versão não fornece protecção contra atacantes que observem todas as linhas da Internet ou controlem a primeira e a última mistura.

Pelas seguintes razões queremos oferecer o sistema antes da versão final:

  • Queremos ganhar experiência com um sistema a funcionar com o objectivo de o poder melhorar.
  • Para sermos capazes de oferecer um serviço seguro, precisamos de um grupo anónimo, estável e grande desde o princípio. Esperamos já ter ganho esse grupo e de sermos capazes de o manter durante o processo de desenvolvimento.
